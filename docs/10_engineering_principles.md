# Naam - Engineering Principles

## Version

v1.0

---

# Philosophy

Engineering decisions should optimize for clarity, maintainability and long-term evolution.

---

# Core Principles

## Build Small

Small services.

Small agents.

Small functions.

---

## Single Responsibility

Every module should have one responsibility.

---

## AI Decides

Software Executes.

AI proposes.

The Execution Engine performs.

---

## Repository Pattern

Database interactions belong only inside repositories.

---

## Document Before Building

Major architectural decisions are documented before implementation.

---

## Testability

Every component should be independently testable.

---

## Observability

Every important action should produce logs.

Every failure should be traceable.

---

## Simplicity

Prefer readable code over clever code.

---

## Long-Term Thinking

Every design decision should remain valid when Naam serves thousands of families.

---

# Coding Standards

PEP-8

Type hints

Docstrings for public functions

Meaningful variable names

Consistent naming conventions

---

# Git Workflow

Feature branches

Pull Requests

Code Reviews

Semantic commits

---

# Engineering Goal

The codebase should be understandable by a new engineer within one day.
